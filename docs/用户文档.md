# 基于 OpenHarmony 的家居设备智能控制系统——用户文档

| 文档属性 | 内容 |
| --- | --- |
| 文档版本 | V1.2 |
| APP 平台 | HarmonyOS 6.1.1 / API 24 |
| 网关平台 | Windows 10/11 或 Linux |
| 更新日期 | 2026 年 7 月 |

## 一、产品简介

本系统通过 HarmonyOS/OpenHarmony 生态 APP 控制独立虚拟网关中的家居设备。APP 与网关使用真实 HTTPS/WSS 通信，网关负责设备状态、自动化规则和审计；APP 不在本机直接模拟设备成功结果。

当前网关预置 15 台多房间演示设备，覆盖灯光、环境监测器、空调、加湿器、浴霸、门锁和窗帘七类驱动。启动后实际显示的设备、房间与在线状态以网关权威快照为准，不应依赖固定数量。

完整的设备类型、控制能力和二维码可注册范围见[设备接入与驱动说明](设备接入与驱动说明.md)。

## 二、运行前准备

### 2.1 APP

- 使用 DevEco Studio 和 HarmonyOS SDK API 24 构建或安装 HAP。
- APP 需要 `ohos.permission.INTERNET` 网络权限。
- 手机或模拟器必须能够访问运行虚拟网关的电脑。

在 `DevControl` 目录执行：

```powershell
.\scripts\build_app.ps1
```

构建签名以本机 `build-profile.json5` 的 DevEco 配置为准。签名可用时产物为 `entry\build\default\outputs\default\entry-default-signed.hap`；未配置签名时才会输出 `entry-default-unsigned.hap`。签名路径、口令、私钥和证书材料均不得提交或写入文档。

### 2.2 虚拟网关

网关项目位于与 APP 工程相邻的 `VirtualGateway` 文件夹，需要 Python 3.11 及以上版本。以下网关命令都在 `VirtualGateway` 目录执行。

安装依赖：

```powershell
cd ..\VirtualGateway
python -m pip install -r requirements.txt
```

生成演示 CA 和网关证书：

```powershell
.\scripts\generate_demo_certs.ps1 -HostName localhost -IpAddress 127.0.0.1
```

脚本生成 `certs/gateway.crt`、`certs/gateway.key`，并把公开 CA 复制到相邻 APP 工程的 `entry/src/main/resources/rawfile/demo_ca.crt`。私钥已被忽略，禁止提交。生成或更换证书后必须重新构建 APP。

启动网关：

```powershell
.\scripts\run_gateway.ps1
```

网关缺少证书或私钥时会拒绝启动，不会自动回退到不安全的 HTTP。

### 2.3 网络与证书

- 默认端口为 `8443`。
- 真机连接时请填写电脑的局域网地址，例如 `https://192.168.1.10:8443`，不要使用手机自身的 `127.0.0.1`。
- 网关证书的主机名或 IP 必须与 APP 输入的地址一致。
- APP 使用打包的 DevControl 演示 CA 验证 HTTPS/WSS，并继续执行 SAN 主机名或 IP 校验；不会关闭证书校验。
- 真机演示时应按电脑局域网 IP 重新生成证书，然后重新构建 APP。

## 三、首次配对

1. 启动虚拟网关。
2. 在网关终端找到六位一次性配对码。配对码有效期为五分钟，并在配对成功或到期后立即更换。
3. 打开 APP 的安全配对页，输入 HTTPS 网关地址和配对码。
4. 点击配对/连接。
5. APP 显示在线后，会自动加载当前全部预置设备的权威快照。

同一来源连续五次输入错误配对码后，网关会锁定十分钟。

配对成功后，APP 使用 HUKS 保护凭据和会话数据密钥。正常关闭并重新打开 APP 时，只要凭据未到期且网关进程未重启，就可以恢复会话，无需再次输入配对码。凭据默认有效 24 小时；到期或网关进程重启后需要重新配对。

## 四、页面功能

### 4.1 控制中心

控制中心展示：

- 网关连接状态；
- 在线设备数量和最近状态；
- 离线快照提示；
- 命令处理中和失败原因；
- 离家模式入口。

离家模式会关闭全部灯光、空调、浴霸和加湿器，关闭全部窗帘并锁定入户门；环境传感器继续监测。只有在线时才能执行，执行后页面逐设备显示成功或失败结果。

### 4.2 照明中心

- 使用开关控制客厅主灯。
- 使用亮度滑块设置 0～100 亮度。
- 开启自动照明后，可设置照度阈值和无人延时。
- 当检测到有人且环境较暗时自动开灯；持续无人达到延时后自动关灯。
- 手动操作后网关会短暂暂停自动覆盖，避免灯光立即被规则改回。

### 4.3 温湿度与空调中心

- 查看实时温度和湿度。
- 控制空调电源。
- 选择制冷、制热或除湿模式。
- 设置 16～30℃ 的目标温度。

环境值由网关连续模拟，温度和湿度会逐步变化。海尔、格力、美的均为模拟品牌适配，不代表真实厂商协议兼容。

### 4.4 智能门禁

- 点击锁门可直接发送上锁命令。
- 点击解锁会弹出二次确认；取消确认不会发送命令。
- 解锁成功后，网关会在约十秒后自动上锁。
- 操作失败、超时或设备离线时，页面显示错误原因。

门锁记录属于安全审计数据，家庭用户不能在 APP 中清空。

### 4.5 智能窗帘

- 从控制中心的“窗帘”入口或客厅设备列表打开智能窗帘。
- 使用开合度滑块设置 0～100 的目标位置，0 表示完全关闭，100 表示完全打开。
- 使用打开、暂停、关闭按钮控制运动；位置由网关逐步推进并通过 WSS 回传。
- APP 不会在命令发送后直接伪造位置，只有收到更高 `stateVersion` 的权威状态才更新。
- 网关离线、窗帘离线或命令处理中时，所有控制入口都会禁用。

### 4.6 设置与诊断

- 查看网关地址、连接状态、协议版本、证书信任方式和最后同步时间；
- 手动重新连接或清除安全会话后重新配对；
- 查看最近五条脱敏命令审计，只显示 messageId 后缀；
- 查看协议错误码的中文解释。

### 4.7 通过二维码添加与删除设备

- 首页右上角“+”会展开菜单，可选择“添加设备”或“添加房间”。“扫描二维码”暂标为“敬请期待”，当前设备接入使用“从图库选择二维码”。
- 二维码携带设备序列号、名称、类型、功能分类、完整能力集合及网关签发的设备身份证书，不携带所属房间。识别后请在“添加至房间”下拉菜单选择当前已有房间；APP 只做格式检查，已配对网关才会验证身份、能力、所选房间和在线性。
- 当前可扫码新增灯光、环境监测器、空调和加湿器。成功后 APP 会同步权威快照，约 1.5 秒后返回首页并自动选中所选房间。
- 二维码本身可长期保存；网关重启后失效的是 APP 配对会话，而不是二维码。重新配对后可继续使用同一张有效二维码。
- 扫码新增的设备可在设备详情页删除；内置演示设备不提供删除入口。删除后可再次扫描同一二维码重新添加。

二维码生成、字段及支持范围见[设备接入与驱动说明](设备接入与驱动说明.md)。

### 4.8 添加房间

- 在右上角“+”菜单选择“添加房间”，输入不重复的房间名称，例如“书房”。
- 房间由已配对网关创建；成功后会显示在首页房间标签和“添加至房间”下拉菜单中。
- 系统默认已添加客厅、主卧、次卧和浴室，且它们会保留在首页顶部标签中；预置设备已归属到这些房间。
- 新建空房间可以先创建、后添加设备；内置“入户”属于门锁安全区域，不作为设备添加目标。当前不支持删除房间，因此不会级联删除房间内设备。

### 4.9 首页导航、总控与房间

- 顶部标签提供“全屋”和已发现设备所在的房间；房间数量变化时，标签会随设备数据自动变化。“设置”位于底部导航栏。
- “总控”展示全屋在线、温湿度、亮灯等概览，并按灯光、窗帘、环境、门锁进入相应功能。门锁是全屋安全功能，只在总控中出现。
- 选择房间标签后，可看到该房间的各个设备；点击任意设备会进入完整控制页面。
- 无论从总控功能、房间列表还是设备详情进入，同类设备使用相同控制能力：灯光可调开关、亮度、自动化参数；环境页可查看传感器并控制空调；门锁解锁时会出现远程解锁确认。
- 窗帘驱动已提供多房间预置设备，可进行打开、暂停、关闭和 0～100 开合度控制；当前尚不支持通过二维码新增窗帘。

## 五、连接与离线行为

APP 连接状态包括：连接中、在线、离线和已关闭。

发生 WebSocket 断开或网络中断时：

1. APP 保留最后一次设备快照并标记为缓存数据；
2. 所有控制入口被禁用，避免显示伪成功；
3. APP 返回安全配对页，并自动填入上一次使用的网关地址；
4. 用户输入网关当前显示的一次性配对码重新配对，成功后重新获取全量快照，以网关状态覆盖本地缓存。

如果命令超时，APP 会结束“处理中”状态并显示 `COMMAND_TIMEOUT`。不要根据按钮动画判断设备已经执行，最终状态以网关回传为准。

## 六、安全说明

- 传输层使用 HTTPS/WSS 和 TLS 证书验证。
- Native C++ 使用 AES-256-GCM 加密控制载荷。
- 每条指令都有独立随机 nonce、messageId、时间戳和状态版本。
- 网关拒绝被篡改、过期、重放或状态版本陈旧的指令。
- APP 使用 HUKS AES-256-GCM 加密配对材料，Preferences 只保存版本、IV、密文和认证标签组成的加密封装；解密时认证标签由 HUKS 独立校验，不保存明文凭据和数据密钥。
- 网关凭据有明确有效期；到期后 HTTP 和已经建立的 WSS 都会失效，APP 会清除旧会话并要求重新配对。
- 日志会对凭据、密钥、nonce 和敏感载荷进行脱敏。

## 七、常见问题

### Q1：APP 无法连接网关

依次检查：

1. 网关进程是否仍在运行；
2. 电脑和设备是否网络互通；
3. 地址是否使用 `https://` 和正确的 `8443` 端口；
4. 防火墙是否允许入站访问；
5. 证书 SAN 是否包含输入的主机名或 IP；
6. 设备是否信任签发网关证书的 CA。

### Q2：配对码无效

配对码只有五分钟有效期，成功使用后也会立即轮换，并且连续失败会触发锁定。请查看网关终端或本机维护接口中的当前配对码，并确认没有输入空格。

### Q3：APP 重启后为什么仍要求配对

如果只是 APP 重启，HUKS 会尝试恢复未过期会话。如果凭据已到期，或网关进程重启导致内存会话丢失，旧凭据会失效，需要重新配对。

### Q4：控制后状态没有立即变化

指令需要经过 Native 加密、WSS 发送、网关校验、设备执行和状态回传。网络异常或状态冲突时可能失败；以页面上的最终状态和错误提示为准。

### Q5：能否连接真实海尔、格力或美的空调

不能。当前三种品牌适配器只模拟不同的命令编码方式，用于展示扩展架构。

## 八、演示建议

推荐按以下顺序演示：

1. 启动网关并展示一次性配对码；
2. APP 配对并加载设备快照；
3. 灯光开关、亮度及自动照明；
4. 温湿度变化与空调模式；
5. 门锁二次确认和自动上锁；
6. 离家模式；
7. 中断网络，展示离线快照、返回配对页和上次网关地址记忆；
8. 重启 APP，展示 HUKS 会话恢复；
9. 展示篡改密文测试被网关拒绝。

## 九、使用边界

本系统是竞赛级软件原型，当前使用软件模拟设备，不能直接用于真实住宅门禁或家电控制。真实部署还需要设备身份、账号权限、硬件安全、生产证书、隐私合规、渗透测试和厂商协议授权。
