安全负向覆盖：

- 系统信任库不接受演示私有 CA；
- 合法 CA 可以完成 HTTPS/WSS；
- 密文篡改、AAD 篡改、超过 30 秒时间戳、复用 nonce 被拒绝；
- 重复 messageId 返回缓存结果且不重复执行；
- 无效 Bearer credential 不能建立有效事件会话。
- 凭据在 `expiresAt` 边界立即失效；已建立的 WSS 以 4401 关闭，后续 REST 返回 `AUTH_FAILED`。

## 构建与交付

- ArkTS、Native C++、资源编译和 HAP 打包通过。
- 仓库默认不含签名配置，因此日常脚本生成无签名 HAP。
- 当前工作区未保留签名 HAP；需要安装到真机或执行 ohosTest 时，应使用被忽略的本地签名配置重新构建并用 SDK `verify-app` 复核包签名、摘要和 Native 代码签名。
- 开发调试 profile 生成的签名包只用于本地安装和竞赛演示，不作为商店发布签名。
- 私钥、数据库、报告和本地签名配置均被忽略。
